# Week 5 - Git (part 3 of 3)

Date

3 October 2024

Agenda

  • Generate and add SSH key to GitHub
  • Review and fixing things in Git

# 1. Checking for existing SSH keys

Before you generate an SSH key, you can check to see if you have any existing SSH keys. Full reference here (opens new window)

You can check for existing keys using the following command in your terminal

ls -la ~/.ssh
1

This will list all the files in your .ssh directory if they exist.

By default, the filenames of supported public keys for GitHub are one of the following:

  • id_rsa.pub
  • id_ecdsa.pub
  • id_ed25519.pub

TIP

If you receive an error that ~/.ssh doesn't exist, you do not have an existing SSH key pair in the default location. You can create a new SSH key pair in the next step.

You can then either upload an existing SSH key or generate a new one.

# 2. Generate a new SSH key

To generate a new SSH key, paste the following command in your terminal, and swap in your email address.

ssh-keygen -t ed25519 -C "your_email@example.com"
1

This creates a new SSH key, using the provided email as a label.

Next it will want to place it in new directory called .ssh inside your home directory.

You should see the following:

Generating public/private ed25519 key pair.
Enter file in which to save the key (/Users/your_name/.ssh/id_ed25519):
1
2

Press Enter to save the key in that location.

Next you will be asked to enter a passphrase (password). This is an extra layer of security that is optional. I recommend you to enter a passphrase. You will be prompted to enter it twice.

Your key has now been generated and saved in your .ssh directory. Check to make sure:

ls -la ~/.ssh
1

# 3. Adding your private SSH key to the ssh-agent

First you want to start the ssh-agent in the background with the following command:

eval "$(ssh-agent -s)"
1

You should see something like this with a different pid:

Agent pid 34566
1

Since you are using a macOS that is later than Sierra 10.12.2, you will need to modify your ~/.ssh/config file to automatically load keys into the ssh-agent and store passphrases in your keychain.

First, check to see if your ~/.ssh/config file exists in the default location with the following command:

open ~/.ssh/config
1

If the config file does not exist, create it:

touch ~/.ssh/config
1

Now open it with the previous open command and add the following text in it.

Host github.com
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519
1
2
3
4

Save and close the config file.

Finally, you need to add your SSH private key to the ssh-agent and store your passphrase in the keychain. Use the following command to do so:

ssh-add --apple-use-keychain ~/.ssh/id_ed25519
1

You should see the following:

Identity added: /Users/your_name/.ssh/id_ed25519 (your_email@example.com)
1

You can see the full reference here on GitHub Docs (opens new window)

# 4. Add your public SSH key to your GitHub account

First, you need to copy your public SSH key to the clipboard. You can do so by running the following command:

pbcopy < ~/.ssh/id_ed25519.pub
1

Next, login to GitHub and follow these steps:

  1. In the upper-right corner of any page on GitHub, click your profile photo, then click Settings.
  2. In the "Access" section of the left sidebar, click SSH and GPG keys.
  3. Click New SSH key.
  4. In the "Title" field, add a descriptive label for the new key. For example, if you're using a personal laptop, you might call this key "Personal laptop". I used "Karim's MBP"
  5. For "Key type" select "Authentication Key".
  6. In the "Key" field, paste your public key.
  7. Click Add SSH key.

That's it!

See full reference here on GitHub Docs (opens new window)
