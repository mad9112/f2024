# 1. Platforms

A platform is a term used frequently in technology and can mean different things depending on the context. For us, it will simply mean anything you can build upon. Since this is a Mobile Development course, it will mean one of the following:

  • Apple OSs (iOS / iPadOS / tvOS / watchOS)
  • Google Android
  • Web

# 2. Native and Hybrid Apps

Mobile apps can be divided in two groups: Native and Hybrid. Native apps uses native APIs from the platform they are running on. Hybrid apps are web apps, optimized for mobile, that run in a native app shell inside a native app.

# 3. Software Development Kit

A SDK (Software Development Kit) is a set of software tools that a developer uses to create new software. SDKs that generate software to multiple platforms based in the same code are called cross-platform SDKs.

# 4. Kotlin vs Java

Until May 2017, the officially supported programming languages for Android were C++ and Java. Kotlin is a modern language and combines both OOP and functional paradigms. It compiles to same byte code as Java making it possible to add Kotlin incrementally to an Android app originally made in Java.

Why the switch to Kotlin? Kotlin allows you to be more concise and write fewer lines of code for the same functionality compared to Java or C++. Apps that are built with Kotlin are also less likely to crash, resulting in a more stable and robust app for users.

# 5. Installing Android Studio

Version 2024.2.X (Ladybug) of Android Studio is the latest version which we will be using.

Here's a full installation guid from Google (opens new window).

  1. Go to this link (opens new window) and click on the "Download Android Studio Ladybug" button to download the dmg file for the installation.

  1. When the download finishes, double click on the dmg file to mount it. Once that is done you will see the window below. Drag the Android Studio icon to the Application Folder shortcut.

You can now close the above window and open Android Studio. You can find it in your applications folder or use spotlight cmd+space to launch Android Studio.

# 6. Android Virtual Devices

An Android Virtual Device (AVD) is a configuration that defines the characteristics of an Android device that you want to simulate in the Android Emulator. You will use them to simulate the running of your application in a real device. Not everything can be done in an emulator, though. For some things you will need a real device.
