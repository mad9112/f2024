7 November 2024

Agenda

# 1. iOS App

Currently there are 2 ways of building apps for iOS in the context of UI creation and use: declarative and imperative.

# 2. Declarative UI

A declarative UI, basically, describes what elements are part of the user interface. Some of the details can be left out, e.g,: style and exact position of the elements. SwiftUI is a declarative way to build interfaces in iOS apps. It is easy to learn, clean and provides simple tools to implement reactive programming. There is a drawback though: It is supported only on iOS13 and above. So, if you want your app to run in an older iOS version, SwiftUI may not be the best option for you.

What iOS version is on iPhones out there?

Apple's official App Store developer's page has a great breakdown on the use of its operating systems.

Check it out (opens new window)

# 3. Imperative UI

An imperative UI, by contrast has to explicitly define all details on style and position. UIKit is the library we use to create imperative UI.

Another distinction that you should be familiar with is the one between Reactive and Imperative programming.

Imperative programs use statements to change the app's state. One of the most used patterns in Imperative programming is the Model-View-Controller.

Reactive programs, differently, uses data streams, propagation of change and bindings.

Checkout this video (opens new window) for a good explanation on declarative vs imperative programming.

Question

So, which should you learn and which should I use?

Answer

If there's a declarative available, you should almost always use it. You will achieve your goal much faster with a lot let headache.

Question

What about the iPhones that are running an iOS version lower than 13?

Answer

If we open Apple's App Store developer's page (opens new window), we can see that as of June 9, 2024 of all the phones introduced since 2020, 97% of them are running an iOS version that is higher than 13. Also, if we look at Apple's iPhone User Guide (opens new window) from their support website, we can see that iOS 13 is supported on iPhones as far back as iPhone SE 1st generation which came out in 2016.

Now that we know that most iPhones in the world are running an iOS 13 or higher, let's move forward with SwiftUI.
